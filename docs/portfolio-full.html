EduLens AI — Problem & Users

Problem

Teachers spend 2–4 hours marking a single worksheet for 35 students. Students receive ticks and crosses but rarely get structured feedback on why they made mistakes or what to practise next. Generic AI chatbots give inconsistent free-form answers unsuitable for mistake books, dashboards, or printable reports.

Solution

EduLens returns validated, structured JSON — marking-style feedback, mistake type tags, similar practice questions, syllabus-aware lesson packs, and exportable PDF/DOCX/PPTX reports. Teachers remain in the loop via teacherReviewRecommended flags.

Target Users

  • Students (13–18): Understand homework mistakes; build a mistake book
  • Teachers / tutors: Draft lesson materials faster with syllabus RAG
  • DSA reviewers: Evaluate AI EdTech safely via mock mode — no API keys exposed

EduLens AI — Core Features

1. Homework Analyzer

Text paste or image upload (vision OCR). Returns per-question scores, mistake types (conceptual, calculation, careless, incomplete, method), improvement tips, and similar questions.

Live: /edulens/homework-analyzer Upload a worksheet photo or paste text → structured feedback cards → mistake book entry

2. Lesson Generator

3-part generation pipeline with staged loading UX. Outputs lesson plan, worksheet, answer key, visual learning steps, and subject-aware diagrams. User selects deliverables (plan only, worksheet, etc.).

Live: /edulens/lesson-generator Enter topic + syllabus level → multi-stage progress → shareable report link

3. Mistake Book & Dashboard

Persistent local history groups errors by type and knowledge point. Dashboard aggregates usage stats for self-review.

EduLens AI — Technical Details & Learning Value

Multi-Provider LLM Chain

Unified completeJSON() interface with ordered fallback: Agnes → Gemini → Groq → OpenRouter → Mock. Vercel demo uses EDULENS_AI_MODE=mock for zero-cost, deterministic seeded responses.

Structured JSON Pipeline

  1. Build prompt with RAG context
  2. LLM completion → extract JSON
  3. Zod validate → apply safety defaults
  4. Enrich (visual lessons, media, assessment alignment)
  5. On failure: compact prompt → next provider → mock

Quality Engineering

  • ~90 library modules under src/lib/edulens/
  • Vitest unit tests: prompt builder, JSON parser, RAG retrieval, visual type detection
  • RAG citation sanitization; subject-aware diagram rendering
  • 3-part lesson generation respects token limits

CampusBot AI — Scenario & Algorithm

CampusBot simulates how a service robot could navigate a school campus — delivering worksheets, guiding visitors, or returning library books. It demonstrates algorithmic thinking and robotics interest without requiring hardware in the DSA review.

Task Scenarios

  • Worksheet delivery: Robot picks up materials from staff room, navigates to classroom
  • Visitor guide: Escort from reception to destination with obstacle avoidance
  • Library return: Collect items and navigate back to library

A* Pathfinding Engine

Grid-based school map with walls, doors, and dynamic obstacles. A* finds optimal path → step simulation → on collision, dynamic re-plan → metrics report.

Algorithm Flow

  1. Parse task goal cell from scenario definition
  2. Build cost grid from map (walls = infinite cost)
  3. Run A* with Manhattan heuristic
  4. Simulate step-by-step movement
  5. On blocked cell: invalidate path segment, re-plan from current position
  6. Output metrics: total steps, collisions, re-plans, success/fail
Live: /campusbot/simulator Select scenario → watch robot navigate → view metrics report

YouthMentor AI — Safety-First Wellbeing Design

YouthMentor addresses student exam stress and emotional pressure — but unlike engagement-optimised AI companions, it is designed with hard safety boundaries and structured reflection rather than open-ended chat.

Core Flow

  1. Mood check-in + stress source selection (+ voice input option)
  2. Choose mentor persona
  3. Six guided reflection prompts (CBT-inspired framework)
  4. Safety pipeline runs before any coaching response
  5. Normal risk → mentor JSON response + action plan
  6. High risk → crisis panel only (helplines, no coaching LLM)
Safety design principle: Never rely on the LLM alone for crisis detection. Keyword scan runs first; high-risk input blocks before any coaching prompt is sent.

Safety Pipeline

LayerAction
1. Keyword scanDetect self-harm / crisis language
2. Risk classifierScore remaining input
3. Crisis-only UIHelplines and resources — zero coaching LLM

YouthMentor AI — Features & Responsible Design

Additional Features

  • Mood trend chart: Visualise intensity from saved reflection history
  • Voice input: Web Speech API on check-in, reflection, character chat
  • Trust metrics: Home page compares responsible design vs engagement-first AI companions
  • Character chat: Empathy-first protocol; labeled AI simulation (not real persons)
  • Privacy: All data in browser localStorage — no account, no cloud storage

What YouthMentor is NOT

  • Not therapy, diagnosis, or emergency care
  • Not official communication from any real person or faith leader
  • Not an engagement-optimised companion designed to maximise chat volume
